- [x] **באג**: עמוד החומש הציג תמיד את אותו טקסט (מ-Sefaria calendars API, שמחזיר טווח אחד לכל השבוע) בלי קשר ליום שנבחר. תוקן ע"י שימוש בשדה `extraDetails.aliyot` שמחזירה Sefaria (טווח פסוקים מדויק לכל עלייה 1–7), ובניית רפרנס דינמי לפי יום בשבוע (יום שישי = מיזוג עליות ו׳–ז׳, שבת = כל הפרשה)

## שלב 36 - תיקון באג אמיתי: TextViewer לא מרענן תוכן בעמוד חומש בבחירת יום

**תוכנית (נכתב לפני ביצוע).** המשתמש דיווח: לוחצים על יום בלוח בחומש/תניא — התוכן לא מתעדכן לאותו יום (בתהילים כן עובד). אומת ב-preview חי עם מעקב network+DOM, לא רק לפי תיאור:

- [x] **אובחן באג אמיתי ומדויק בחומש**: לחיצה על יום אחר (12→20) עדכנה נכון את הכותרת ("עלייה א׳"→"עלייה ב׳") אבל **אף בקשת fetch חדשה לא נשלחה ל-Sefaria**, והפאנל נשאר פתוח וריק (לא "טוען", לא שגיאה — פשוט תקוע). שורש הבעיה ב-`TextViewer.jsx`: שני `useEffect` נפרדים — האחד מאפס `sections`/`error`/`open` כש-`sefariaRef` משתנה, השני מבצע fetch רק אם `!open || sections || error || loading`. שני ה-effects רצים **באותו commit** על סמך אותו רינדור — effect ה-fetch קורא את `sections` הישן (עוד לפני שה-`setSections(null)` מה-effect הראשון "נקלט"), רואה שהוא not-null, ומדלג על ה-fetch. ברינדור הבא, `sections` כבר null — אבל הוא **לא ברשימת התלויות** של effect ה-fetch (`[open, sefariaRef]` בלבד), כך שה-effect לא רץ שוב. תוצאה: תוכן קפוא מהיום הקודם שנבחר, לצמיתות, עד שינוי חיצוני אחר (כמו ניווט/reload) יפעיל fetch מחדש
- [x] **הוסבר למה תהילים "עובד"**: ב-`Tehillim.jsx`, ה-`<TextViewer>` עטוף ב-`<div key={entry.day}>` — כש-`entry.day` משתנה, React מפרק ובונה מחדש את כל תת-העץ (remount מלא), כך שה-state הפנימי (`sections`) מתחיל מ-null אמיתי בלי הבאג להתגלות. `Chumash.jsx` **אין** לו key כזה בכלל. `Tanya.jsx` יש לו key בענף היום-הלא-נוכחי (ולכן נבדק ועבד נכון ב-3 תרחישים), אבל ענף "היום" (Tanya Yomi החי) אין לו — לא באג פעיל היום, אבל שביר ולא מכוון
- [x] **תיקון בשורש** (לא עוקף עם key): מיזוג שני ה-effects ב-`TextViewer.jsx` לאחד — אותו effect שמאפס (`setSections(null)`, `setError(false)`) הוא זה שגם מבצע את ה-fetch, על סמך `[open, sefariaRef]` בלבד, בלי תלות ב-state שהוא עצמו קורא לפני שהתעדכן. מתקן את הבאג בשלושת העמודים במקור אחד, לא רק "מטשטש" אותו דרך key
- [x] אימות ב-preview: חומש — לחיצה בין ימים מרובים מרעננת תוכן נכון בכל פעם (לא רק כותרת); תהילים ותניא — עדיין עובדים נכון אחרי השינוי (לא נשבר כלום)
