# WORKPLAN - חת"ת יומי

## מבוא

תוכנית העבודה מסודרת בשלבים כך שכל שלב נשמר ב-git עם commit ברור, ושולחים עדכונים תדירים של ההתקדמות.

## שלב 1 - תוכניות ואפיון ראשוני

- הכנת PRD מלא ומסמכים ראשוניים
- קבלת אישור על דרישות מרכזיות
- הגדרת MVP מפורט
- קביעת כלי הפיתוח: React + Vite + React Router

## שלב 2 - סביבה וסקלטון בסיסי

- יצירת קבצי הפרויקט הראשוניים
- הגדרת `package.json`, `vite.config.js`, `index.html`
- הגדרת `src/App.jsx`, `src/main.jsx`, `src/index.css`
- הוספת ניווט בין דפים
- commit ושמירה ב-git

## שלב 3 - לוח לימוד יומי

- פיתוח `DailyBoard` עם בחירת יום בחודש
- הצגת חומש, תהילים ותניא ליום הנבחר
- יצירת מודל נתונים ראשוני ב-`src/data/dailyContent.js`
- שמירת בחירת משתמש ב-localStorage
- commit ושמירה ב-git

## שלב 4 - מערכת ניקוד רצף

- פיתוח חישוב ניקוד על חומר יומי שנסומן כמושלם
- שמירת סטטיסטיקת רצף ב-localStorage
- הצגת ניקוד מצטבר וסטיקרים
- commit ושמירה ב-git

## שלב 5 - חידון פרשת השבוע

- יצירת מסך חידון `Quiz`
- יצירת שאלות מבוססות רמות קושי
- חישוב ניקוד תוצאה ושמירה פנימית
- commit ושמירה ב-git

## שלב 6 - עיצוב וחוויית משתמש

- עיצוב ממשק בעברית עם טאצ' מודרני
- שיפור קומפוננטות, חוצצים ותצוגות
- תמיכה במסכים ניידים
- commit ושמירה ב-git

## שלב 7 - תזכורות והגדרות

- פיתוח מסך הגדרות תזכורות
- שמירת זמן ותזכורות ב-localStorage
- יצירת התראה פנימית בעת כניסה לאפליקציה
- commit ושמירה ב-git

## שלב 8 - בדיקות ושיפור ביצועים

- בדיקת UI ותיקון באגים
- בדיקות ידניות של זרימת היום, רצף וחידון
- שיפור מהירות הטענות
- commit ושמירה ב-git

## שלב 9 - סיכום והעלאה

- רענון README עם הוראות
- בוודאי שהקבצים נשמרו בכל שלב ב-git
- בדיקת `git status` ו-push אחרון

## ניהול גרסאות והתחייבות

- כל שלב משנה יופיע ב-commit נפרד
- Keep working tree clean as possible
- push ל-origin אחרי כל שלב מהותי
