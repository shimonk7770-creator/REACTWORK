# PRD - חת"ת יומי

## מטרה ובעיה

**מה בונים:** אפליקציית React בעברית לניהול לימוד "חת״ת" יומי (**ח**ומש, **ת**הילים, **ת**ניא) —
מנהג לימוד יומי הנהוג בין לומדי חסידות חב"ד — עם חידון פרשת השבוע, ניקוד רצף ותזכורות.

**הבעיה שבה מטפל הפרויקט:** לימוד חת"ת יומי דורש מהמשתמש לדעת בכל יום איזה קטע ללמוד
בכל אחד משלושת הספרים (חומש לפי יום בשבוע, תהילים ותניא לפי היום העברי בחודש/בשנה),
לאתר את הטקסט המדויק, ולשמור על רצף לימוד לאורך זמן. ללא כלי ייעודי, זה דורש חיפוש
ידני במספר מקורות בכל יום ואין דרך פשוטה לעקוב אחר התמדה. האפליקציה פותרת זאת ע"י
חישוב אוטומטי של תוכן היום לפי הלוח העברי, שליפת הטקסט המדויק מ-Sefaria, ומעקב רצף/ניקוד
שמעודד המשך לימוד.

## מטרות עסקיות

- ✅ לאפשר למשתמש ללמוד תוכן יומי באופן פשוט ומעוצב
- ✅ לשפר שמירת רצף לימוד באמצעות ניקוד ומוטיבציה
- ✅ לתת חוויית חידון מעשירה על פרשת השבוע
- ✅ לשמור את כל השינויים והתקדמות המשתמש באופן מקומי ובגיט

## קהל יעד ותרחיש שימוש

**קהל יעד:**
- לומדי חומש, תהילים ותניא יומי
- קהל דתי המעוניין בכלי תרגול יומי וניקוד רצף
- משתמשים שאוהבים חוויית לימוד כיפית ועיצוב מודרני

**תרחיש שימוש טיפוסי:**
1. המשתמש נכנס לאתר בבוקר (או מקבל תזכורת פנימית בשעה שקבע מראש בהגדרות).
2. בדף הבית רואה מיד סיכום "מה לומדים היום" בשלושת הספרים, וסרטון/מחשבה יומית.
3. לוחץ על אחד מכפתורי הנושא (חומש/תהילים/תניא) — מגיע לעמוד הייעודי עם הטקסט המדויק
   של אותו יום, כבר פתוח וטעון (ללא צורך בחיפוש).
4. לאחר הלימוד, לוחץ "סמנתי כנלמד" — מקבל ניקוד, והרצף שלו מתעדכן (עם בונוסים על אבני דרך).
5. יכול גם לגלוש לימים אחרים בלוח השנה העברי (למשל אם החסיר יום), ולראות את המפה
   השנתית של ימי הלימוד שלו בהגדרות.
6. פעם בשבוע, בודק את עצמו בחידון פרשת השבוע ובוחר רמת קושי מתאימה.

## היקף (Scope)

### בתוך ההיקף
- 3 עמודי לימוד יומי (חומש / תהילים / תניא) מסונכרנים ליום העברי, עם לוח שנה עברי לחיץ
- חידון פרשת השבוע עם 3 רמות קושי ומעקב שאלות-שנשאלו
- מערכת ניקוד ורצף (streak) הנשמרת מקומית
- תזכורת יומית פנימית (באנר בתוך האפליקציה)
- דף בית, דף הגדרות, ותיאום תוכן מ-Sefaria API
- PWA בסיסי — manifest + service worker network-first, ניתן להתקנה למסך הבית

### מחוץ להיקף (Out of scope)
- **ללא backend / בסיס נתונים** — כל המצב נשמר ב-`localStorage` בדפדפן בלבד; אין סנכרון בין מכשירים/דפדפנים
- **ללא התחברות משתמשים** — אין הרשמה, התחברות, או תמיכה במספר פרופילים על אותו מכשיר
- **ללא התראות Push אמיתיות** — התזכורת מוצגת רק כשהאפליקציה פתוחה בדפדפן, לא כהתראת מערכת הפעלה (למרות שיש PWA בסיסי — ר' "בתוך ההיקף" — אין לו push notifications)
- **ללא Offline מלא** — טעינת תוכן לימוד (חומש/תהילים/תניא) דורשת חיבור אינטרנט ל-Sefaria; ה-PWA (manifest + service worker network-first) מאפשר התקנה למסך הבית וטעינה חוזרת של קבצים סטטיים, אך לא עבודה מלאה ללא רשת
- **ללא הרחבת בנק שאלות החידון** מעבר למקור הקיים (abc770.org) — נבדק מקור חלופי (shayleyeladim.org.il) ונמצא לא רלוונטי (ר' "מקורות ונתונים")
- **ללא תמיכה רב-לשונית** — האפליקציה בעברית בלבד, RTL קבוע

## בחירות טכנולוגיות (Tech Stack)

| טכנולוגיה | גרסה | סיבת הבחירה |
|---|---|---|
| **React** | 18 | ספריית UI מובילה, hooks, קומפוננטים לשימוש חוזר |
| **Vite** | 6 | Build tool מהיר עם Hot Module Replacement — פיתוח נוח |
| **React Router** | v6 | ניווט client-side ללא טעינת דף, SPA אמיתי |
| **JavaScript ES2022** | — | פשטות — ללא TypeScript בשלב ה-MVP |
| **CSS3 ידני** | — | שליטה מלאה בעיצוב RTL ללא תלות בספריית UI חיצונית |
| **localStorage** | Web API | שמירת מצב מקומי ללא צורך ב-backend או DB |

### ספריות חיצוניות שנבחרו **לא** להשתמש בהן (ומדוע)
- **Redux / Zustand** — המצב קטן ומקומי, `useState` + `localStorage` מספיקים
- **Tailwind / MUI / Chakra** — עיצוב RTL עברי דורש שליטה מלאה ב-CSS
- **Axios / React Query** — אין API חיצוני בשלב הראשון
- **TypeScript** — MVP מהיר; ניתן להוסיף בהמשך

### אילוצים (Constraints)
- **RTL עברי מלא** — כל העיצוב חייב לתמוך בכיוון ימין-לשמאל; זו הסיבה המרכזית לפסילת ספריות UI מוכנות (ר' לעיל)
- **ללא backend** — כל שמירת מצב מוגבלת ל-`localStorage` בדפדפן; אין סנכרון בין מכשירים (ר' Out of scope)
- **תלות ב-Sefaria API** — טעינת טקסטים דורשת רשת וזמינות Sefaria; אין fallback offline לתוכן לימוד (רק לקבצים סטטיים דרך ה-PWA)
- **Autoplay עם קול חסום בדפדפנים** — סרטון הפתיחה חייב להיות מושתק (`muted`) כדי להתנגן אוטומטית, מגבלת מדיניות דפדפנים לא של האפליקציה
- **לוח פרשות סטטי** — מקודד ידנית לשנת ה׳תשפ״ו בלבד; דורש עדכון ידני לשנים הבאות
- **בנק שאלות חידון מוגבל** — ~15–36 שאלות לפרשה בלבד (תלוי בפרשה), ממקור סטטי יחיד (abc770.org)

### ניהול מצב (State Management)

```
localStorage
├── reactwork-daily-state              { completedDays[], streak, score, lastCompletedDate,
│                                         history[], selectedDateISO }
│                                        — completedDays מפתח לפי יום עברי בחודש (1–30), משותף לשלושת העמודים
│                                        — history: כל תאריכי ה-ISO (מקומיים) שסומנו אי-פעם, לא מתאפס
│                                          מדי חודש (בשונה מ-completedDays) — מוצג במפת הרצף השנתית
├── reactwork-settings                 { remindersOn, reminderTime, fontSize, theme }
├── reactwork-city                     string (שם עיר לחישוב זמני שבת/חגים, ברירת מחדל: Jerusalem)
├── reactwork-quiz-best:<רמה>          number (שיא ניקוד לכל רמת קושי בנפרד)
├── reactwork-quiz-seen:<פרשה>:<רמה>   number[] (אינדקסים של שאלות שכבר נשאלו בפרשה+רמה זו)
└── reactwork-reminder-seen            string (תאריך)
```

**גישה:** `useDailyProgress()` הוא הוק משותף בין שלושת עמודי הלימוד — כך שסימון "נלמד"
בכל אחד מהם משפיע על אותו רצף/ניקוד יחיד. חוץ מזה, כל עמוד מנהל את המצב שלו עם
`useState` + `useEffect` לקריאה/כתיבה ב-`localStorage`. אין state management גלובלי.

### ישויות מרכזיות (Core Entities)

| ישות | שדות עיקריים | מקור |
|---|---|---|
| **יום לימוד (Daily Entry)** | `dayOfWeek`/`hebrewDayOfMonth`, `chumashRef`, `tehillimRef`, `tanyaRef`, `note` | `dailyContent.js` (מקומי) + Sefaria (רפרנס חי) |
| **פרשה (Parasha)** | `name`, `book`, `date`, `aliyot[]` (טווח פסוקים לכל עלייה) | `parashaData.js` + Sefaria `extraDetails.aliyot` |
| **שאלת חידון (Quiz Question)** | `q` (טקסט), `a` (4 אפשרויות), `correctIndex`, `parasha` | `parashaQuestions.js` |
| **התקדמות (Progress)** | `completedDays: Set`, `history: string[]`, `streak`, `score`, `lastCompletedDate` | `useDailyProgress` ↔ `reactwork-daily-state` |
| **הגדרות משתמש (Settings)** | `remindersOn`, `reminderTime`, `fontSize`, `theme`, `city` | `reactwork-settings` / `reactwork-city` |

### זרימת נתונים (Data Flow)

```
dailyContent.js (static, מפתח: יום עברי)          parashaQuestions.js (בנק שאלות)
        ↓                                                    ↓
hebrewDate.js: getHebrewDayInfo(selectedDate)        Quiz.jsx ←→ localStorage (bestScore, seen)
        ↓
useDailyProgress() ←→ localStorage (streak, score, completedDays)
        ↓
Chumash.jsx / Tehillim.jsx / Tanya.jsx ←→ HebrewCalendarGrid (onSelectDay)
Settings.jsx   ←→ localStorage (reminderTime, remindersOn)

```

## שימוש מרכזי

1. ✅ שלושה עמודי לימוד ייעודיים (חומש / תהילים / תניא):
   - ✅ כל נושא מקבל עמוד, עיצוב וצבע משלו (במקום כרטיס אחד דחוס)
   - ✅ תוכן אמיתי — תהילים ותניא לפי **יום עברי** בחודש (`HDate`, לא לועזי), חומש לפי עלייה מדויקת ליום בשבוע (Sefaria `extraDetails.aliyot`)
   - ✅ חומש מחשב את פרשת ה**שבוע הנכון** לפי התאריך שנבחר בפועל (לא תמיד "פרשת השבוע הנוכחית") — לחיצה על יום ראשון בשבוע הבא מציגה את פרשת השבוע הבא, לא את של השבוע הזה
   - ✅ יום כ״ט/ל׳ מוצג מאוחד בחודשים עבריים חסרים (29 יום), כמנהג חב״ד
   - ✅ לוח שנה עברי לחיץ בכל עמוד — לחיצה על יום מציגה מיד את התוכן שלו, **והבחירה נשמרת ועקבית בין שלושת העמודים** (לא מתאפסת במעבר בין דפים)
   - ✅ שם פרשת השבוע מוצג ישירות על תאי השבת בלוח השנה
   - ✅ כרטיס רצף/ניקוד משותף לשלושת העמודים (סימון "נלמד" בכל אחד מהם תורם לאותו רצף)
