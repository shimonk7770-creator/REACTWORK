# PRD - חת"ת יומי

## תמצית

פרויקט "חת"ת יומי" הוא אפליקציית React בעברית לניהול לימוד יומי של חומש, תהילים ותניא, עם חידוני פרשת השבוע, ניקוד רצף ותזכורות.

## מטרות עסקיות

- לאפשר למשתמש ללמוד תוכן יומי באופן פשוט ומעוצב
- לשפר שמירת רצף לימוד באמצעות ניקוד ומוטיבציה
- לתת חוויית חידון מעשירה על פרשת השבוע
- לשמור כל שלב ושינוי ב-git כדי לתעד את ההתקדמות

## קהל יעד

- לומדי חומש, תהילים ותניא יומי
- קהל דתי המחפש כלי לימוד יומי נגיש
- משתמשים שמחפשים מערכת ניקוד ורצף

## שימוש מרכזי

1. לוח לימוד יומי:
   - הצגת חומש, תהילים ותניא לכל יום בחודש
   - בחירת היום בחודש עם עדכון התוכן המתאים
   - שימוש במידע יומי ממקור חיצוני (Chabad) רק עבור התוכן של כל יום

2. מערכת רצף וניקוד:
   - סימון יום כ"מושלם"
   - הוספת ניקוד על ביצוע יום לימוד
   - ניהול רצף יומי וסטטיסטיקה אישית
   - שמירת מצב משתמש ב-localStorage

3. חידון פרשת השבוע:
   - שאלות טריוויה על פרשת השבוע
   - ניקוד לפי רמת קושי
   - הצגת תוצאות ומשוב למשתמש

4. תזכורות:
   - מסך הגדרות תזכורות יומיות
   - שמירת זמן תזכורת
   - התראה פנימית בעת כניסה לאפליקציה

## דרישות פונקציונליות

- מסכי בית, לוח יומי, חידון ותזכורות
- הצגת תוכן יומי לפי יום בחודש
- שמירת התקדמות משתמש בנפרד לכל יום
- ממשק שימוש בעברית עם עיצוב RTL
- שמירה מקומית והיסטוריית שימוש
- שמירה ב-git אחרי כל שינוי חשוב

## דרישות טכנולוגיות

- React + Vite
- React Router
- CSS מודרני לתמיכה ב-RTL
- localStorage לשימור מצב
- קבצי נתונים סטטיים ראשוניים לתוכן היומי

## MVP

- דף בית מעוצב
- לוח לימוד יומי עם חומש, תהילים ותניא ליום
- מערכת ניקוד רצף בסיסית
- חידון פרשת השבוע בסיסי
- שמירת מצב במקומי

## מטרות ראשוניות

1. לייצר ממשק יומי ופיצ'ר רצף
2. לבנות חידון פרשה עם ניקוד
3. לעצב את האפליקציה בעברית ולהיות מוכנים לשינויי תוכן

## שאלות פתוחות

1. האם נשתמש בנתונים סטטיים או מקור חיצוני לצורכי MVP?
2. האם התוכן היומי של חומש, תהילים ותניא יהיה תמיד קבוע לפי יום בחודש?
3. האם רוצים תזכורות רק בדפדפן או גם כאפליקציה נייטיב בעתיד?
4. האם תידרש תמיכה במספר משתמשים או רק שמירה מקומית למשתמש אחד?
